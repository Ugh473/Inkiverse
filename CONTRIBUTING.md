# Contributing to Inkiverse

Thank you for your interest in contributing to Inkiverse! We welcome contributions from writers, developers, designers, and community members.

## Ways to Contribute

### 👨‍✍️ Writers & Content Creators

- **Publish Your Work**: Create and publish comics, novels, light novels, manga, manwha, or manhua
- **Community Reviews**: Provide constructive feedback on other creators' work
- **Guidelines Feedback**: Help improve publishing guidelines and standards
- **Beta Testing**: Test new features and report issues

### 👨‍💻 Developers

- **Feature Development**: Help build new platform features
- **Bug Fixes**: Report and fix bugs
- **Documentation**: Improve code and user documentation
- **Performance**: Optimize platform performance

### 🎨 Designers & Community

- **Design Contributions**: Create UI/UX improvements
- **Community Moderation**: Help maintain community standards
- **Marketing**: Help promote great content

## Publishing Guidelines

### Before Publishing

1. Ensure your content meets our content standards
2. Include appropriate content warnings
3. Use proper formatting for your content type
4. Add metadata (tags, synopsis, cover image)

### Content Types

Each content type has specific formatting requirements. See the relevant guide:

- **Comics**: Panel layout, image formats, and dialogue formatting
- **Novels**: Text formatting, chapter structure, and metadata
- **Light Novels**: Japanese formatting standards and illustration placement
- **Manga**: Japanese reading direction, panel flow, and quality standards
- **Manwha**: Korean formatting standards and vertical reading
- **Manhua**: Chinese formatting standards and publishing conventions

### Quality Standards

- Clear, legible content
- Appropriate metadata and descriptions
- Helpful cover images/thumbnails
- Proper tagging for discoverability
- Original or properly licensed content

## Community Standards

All contributors must follow our [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

### Be Respectful

- Treat all community members with respect
- Provide constructive feedback
- Respect diverse perspectives and cultures

### Keep Content Safe

- Use content warnings appropriately
- Report inappropriate content
- Follow content guidelines
- Respect copyright and intellectual property

## Reporting Issues

Found a bug? Have a feature request?

1. Check existing issues first
2. Provide clear, detailed descriptions
3. Include relevant information (browser, device, etc.)
4. Use issue templates when available

## Development Workflow

### Setup

```bash
git clone https://github.com/Ugh473/Inkiverse.git
cd Inkiverse
npm install
```

### Development

1. Create a feature branch
2. Make your changes
3. Test thoroughly
4. Submit a pull request with clear description

### Pull Request Process

1. Update documentation as needed
2. Add tests for new features
3. Follow code style guidelines
4. Reference related issues
5. Await review from maintainers
